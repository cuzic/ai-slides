## 演習題材の最終決定

### 推奨: 得意先マスター（m021_customer）

#### 選定理由
1. **シンプルな構造**
   - 親子関係なし
   - 基本的なCRUD操作のみ

2. **参考例が豊富**
   - m001_user（ユーザーマスター）が類似
   - 既存コードを参考にしやすい

3. **6時間で完成可能**
   - テーブル設計: 30分
   - 画面設計: 30分
   - 実装: 90分
   - テスト: 60分
   - レビュー・ドキュメント: 30分
   - バッファ: 60分

#### テーブル定義案
```sql
CREATE TABLE m021_customer (
     customerCd NVARCHAR(20) NOT NULL      -- 得意先コード
    ,customerNm NVARCHAR(100) NOT NULL     -- 得意先名
    ,tel NVARCHAR(15)                      -- 電話番号
    ,zipCode NVARCHAR(8)                   -- 郵便番号
    ,address NVARCHAR(200)                 -- 住所
    ,closingDay INT DEFAULT 31             -- 締日（1-31）
    ,baseCd NVARCHAR(2)                    -- 拠点コード
    ,delFlg INT DEFAULT 0 NOT NULL         -- 削除フラグ
    ,regUser NVARCHAR(20)                  -- 登録ユーザー
    ,regTime DATETIME                      -- 登録日時
    ,updUser NVARCHAR(20)                  -- 更新ユーザー
    ,updTime DATETIME                      -- 更新日時
     PRIMARY KEY (customerCd)
);
```

#### 機能範囲
- ✅ 一覧表示（検索機能付き）
- ✅ 新規登録
- ✅ 編集
- ✅ 削除（論理削除）

#### 画面仕様
**一覧画面**:
- 検索条件: 得意先コード（部分一致）、得意先名（部分一致）、拠点（完全一致）
- 表示項目: 得意先コード、得意先名、電話番号、拠点、締日、操作ボタン

**登録・編集画面**:
- 入力項目: 全項目
- バリデーション: 必須チェック、文字数チェック、締日範囲チェック（1-31）

## よくある問題と対策

### 問題1: 環境構築でつまずく
**症状**: インストールエラー、起動しない

**対策**:
1. 事前インストールを必須化
2. Windowsサーバー（共有環境）を予備として用意
3. 5分以上かかるなら共有環境に切り替え

---

### 問題2: AIの生成結果が期待と違う
**症状**: コードが思った通りにならない

**対策**:
1. 必ず参考例（既存コード）を提示
2. 「M001と同じ構造で」と明示的に指示
3. 生成→確認→修正のサイクルを回す
4. ガチャ要素があることを事前に伝える

**良いプロンプト例**:
```
M001UserEntity.java（以下に貼り付け）を参考に、
M021CustomerEntityを作成してください。
テーブル定義は以下の通りです:
[テーブル定義を貼り付け]
```

---

### 問題3: 演習時間が足りない
**症状**: 予定通り進まない

**対策（段階的に適用）**:
1. **30分遅れ**: モックアップを簡素化
2. **1時間遅れ**: 削除機能を省略
3. **1.5時間遅れ**: 編集機能も省略、登録と一覧のみ

**判断基準**:
- 完成度より体験を重視
- 最低限「動くもの」を完成させる

---

### 問題4: 参加者のスキル差
**症状**: 進度がバラバラ

**対策**:
1. 進んでいる人には応用課題
   - 納入先変換マスターに挑戦
   - バリデーションを追加

2. 遅れている人にはサポート強化
   - ヒントを多めに
   - AIへのプロンプト例を提示

3. ペアプログラミング形式も検討
   - スキルのある人と組ませる

## トークン数削減の重要性

### なぜ重要か
- **コスト**: 毎回全ソース読み込むと高額
- **時間**: 読み込みに数分かかる
- **効率**: 無駄な待ち時間が発生

### 解決策
**初回のみ**:
- プロジェクト全体を読み込む（仕方ない）
- 要約README.mdを作成

**2回目以降**:
- README.mdのみ読み込む
- 必要なファイルだけ個別に指定

### 効果
- トークン数: **50,000 → 5,000**（1/10）
- 時間: **3-5分 → 30秒**（1/6）
- コスト: **1/10**

### Day1最後の演習で実施
全員がプロジェクトREADME.mdを作成し、Day2に備える。
