# TDD Plan - テスト駆動開発の計画

現在のタスクに対してTDD（Test-Driven Development）サイクルの計画を立てます。

## あなたの役割

1. **現在のタスクを分析**
   - Todoリストから現在 `in_progress` のタスクを確認
   - タスクの要件と受入条件を理解

2. **テスト戦略を立案**
   - どのテストを書くべきか（単体テスト、統合テスト）
   - テストケースの優先順位付け
   - 各テストケースの期待する動作

3. **TDDサイクルの計画を作成**
   - Red phase: 書くべきテストの具体的な内容
   - Green phase: 実装するべき最小限のコード
   - Refactor phase: リファクタリングの候補

4. **計画を出力**
   - テストケースのリスト（優先順位順）
   - 各テストケースの説明
   - 推定所要時間

## 出力形式

```markdown
### TDD Plan for: [タスク名]

#### Test Cases (優先順位順)

1. **[テストケース名]**
   - 目的: [何をテストするか]
   - 期待する動作: [期待する結果]
   - Red phase: [書くべきテストの概要]
   - Green phase: [実装する最小限のコード]

2. **[テストケース名]**
   ...

#### 推定所要時間
- Red: X分
- Green: Y分
- Refactor: Z分
- 合計: 約W分

#### 次のステップ
`/tdd-red` を実行して最初のテストケースを実装してください。
```

## 重要な原則

- **Small Steps**: 一度に1つの小さなテストから始める
- **Simplest Test First**: 最も単純なテストケースから始める
- **One Test at a Time**: 1つのテストが通ってから次に進む
- **Test Behavior, Not Implementation**: 実装ではなく振る舞いをテストする

現在のタスクを確認してTDDサイクルの計画を立ててください。
